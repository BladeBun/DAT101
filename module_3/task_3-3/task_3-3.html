1. [1. What is the difference between invoking and creating a function?]

    2. [To create a function refers to the process of defining the function using the appropriate syntax. To invoke a function refers to calling the function to actually execute its code.]

  2. [2. Explain the syntax regarding semicolons and code blocks.]

    3. [Semicolons are used to mark the end of a statement. Meanwhile code blocks is a set of statements enclosed by curly braces {}.]

  3. [3. What happens when we try to store values from a function when it doesn't return any?]

    4. [If the function does not have a return statement, it implicitly returns undefined.]

  4. [4. How do we separate parameters from each other when creating a function?]

    5. [When creating a function, parameters are separated by commas which are the placeholders for the values you will pass into the function when you invoke it.]

  5. [5. Explain how parameter names are related during the invocation and creation of a function.]

    6. [Parameter names act as a local variable within the function. The actual value you pass when invoking the function correspond to the parameter.]

  6. [6. Explain why a parameter is undefined when a function call is missing a parameter.]

    7. [If a function call does not include all the arguments for the parameters defined in the function, those parameters will be assigned the value undefined.]

  7. [7. Explain what happens when we use recursive functions.]

    8. [A recursive function is a function that calls itself in its body. For example: factorial(number) is used to compute the result by calling itself.]
